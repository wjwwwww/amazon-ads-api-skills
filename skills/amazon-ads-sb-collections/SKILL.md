# Amazon Ads — Sponsored Brands Collections (SBC) API Integration Guide

## 1. Product Overview

### What are Sponsored Brands Collections?

Sponsored Brands Collections (SBC) is an ad format that enables advertisers to feature **up to 10 related products** in a single ad unit across high-visibility placements on the Amazon Store. It replaces the legacy Product Collections (PC) ad format.

### Collection Types

| Feature | Manual Collections | Automatic Collections |
|---------|-------------------|----------------------|
| Product selection | Advertiser selects 3–10 ASINs | Amazon AI selects up to 10 ASINs dynamically |
| Targeting | Keyword OR Product targeting | Keyword targeting only |
| Landing page | Brand Store or new landing page | Auto-generated landing page |
| Brand logo | Optional | Optional |
| Brand name | Required | Required |
| Headline | Auto-generated by Amazon | Auto-generated by Amazon |
| Multiple ads per ad group | Yes | No (only one ad per ad group) |
| Product edits post-creation | Yes | Only product exclusions (up to 100 ASINs) |

### Key Differences from Legacy Product Collections (PC)

- Up to **10 products** (vs 0–3 in PC)
- **No lifestyle images** — product-centric experience only
- **No advertiser headline** — titles are auto-generated by Amazon AI
- **Branded proof points** — automatically generated to highlight brand popularity
- Products display with key details (title, rating, pricing) for quick evaluation
- Minimum **3 ASINs** required (vs 0 in PC)

### Placements

- Search: Top of Search and Rest of Search (mobile + desktop)
- Up to 3 ASINs displayed per carousel page (may expand to 4 depending on screen size)

### Performance Benchmarks

Early US tests show SBC product-centric experience in top of search delivers:
- **+19% increase in clicks**
- **+44% increase in conversions**

(vs prior lifestyle image-based format)

## 2. API Integration

### 2.1 Authentication & Headers

All requests require:

| Header | Description | Required |
|--------|-------------|----------|
| `Amazon-Ads-ClientId` | Client ID from "Login with Amazon" | Yes |
| `Amazon-Ads-AccountId` | Advertiser Account ID | Optional |
| `Amazon-Advertising-API-Scope` | Profile ID | Optional |
| `Authorization` | Bearer token (OAuth2) | Yes |

**OAuth2 Scopes:** `advertising::campaign_management`

**Regional Endpoints:**

| Region | Base URL |
|--------|----------|
| North America | `https://advertising-api.amazon.com` |
| Europe | `https://advertising-api-eu.amazon.com` |
| Far East | `https://advertising-api-fe.amazon.com` |

### 2.2 Campaign Hierarchy

```
Campaign (costType: CPC, adProduct: SPONSORED_BRANDS)
  └── Ad Group
       ├── Ad (creative: autoCollectionSettings OR manualCollectionSettings)
       └── Targets (keyword, product, productCategory, theme)
```

### 2.3 Create a Campaign

**Endpoint:** `POST /adsApi/v1/create/campaigns`

```json
{
  "campaigns": [
    {
      "adProduct": "SPONSORED_BRANDS",
      "name": "My SBC Campaign",
      "state": "ENABLED",
      "costType": "CPC",
      "marketplaceScope": "SINGLE_MARKETPLACE",
      "startDateTime": "2026-02-01T00:00:00Z",
      "budgets": [
        {
          "budgetType": "MONETARY",
          "budgetValue": {
            "monetaryBudgetValue": {
              "monetaryBudget": {
                "value": 50.00
              }
            }
          },
          "recurrenceTimePeriod": "DAILY"
        }
      ]
    }
  ]
}
```

### 2.4 Create an Ad Group

**Endpoint:** `POST /adsApi/v1/create/adGroups`

```json
{
  "adGroups": [
    {
      "adProduct": "SPONSORED_BRANDS",
      "campaignId": "<campaign-id>",
      "name": "My Collection Ad Group",
      "state": "ENABLED"
    }
  ]
}
```

### 2.5 Create an Ad — Automatic Collection

**Endpoint:** `POST /adsApi/v1/create/ads`

Minimum required inputs: **ad name** and **brand name** (2 fields).

```json
{
  "ads": [
    {
      "adGroupId": "<ad-group-id>",
      "adProduct": "SPONSORED_BRANDS",
      "adType": "COMPONENT",
      "name": "Auto Collection Ad",
      "state": "ENABLED",
      "creative": {
        "componentCreative": {
          "autoCollectionSettings": {
            "sharedSettings": {
              "brand": "Your Brand Name"
            },
            "productExclusions": [
              {
                "productId": "B0XXXXXXX1",
                "productIdType": "ASIN"
              }
            ]
          }
        }
      }
    }
  ]
}
```

**Optional fields for auto collections:**
- `sharedSettings.brandLogos` — brand logo image asset
- `productExclusions` — up to 100 ASINs to exclude

### 2.6 Create an Ad — Manual Collection

**Endpoint:** `POST /adsApi/v1/create/ads`

Minimum required inputs: **ad name**, **brand name**, and **products** (3 fields).

```json
{
  "ads": [
    {
      "adGroupId": "<ad-group-id>",
      "adProduct": "SPONSORED_BRANDS",
      "adType": "COMPONENT",
      "name": "Manual Collection Ad",
      "state": "ENABLED",
      "creative": {
        "componentCreative": {
          "manualCollectionSettings": {
            "sharedSettings": {
              "brand": "Your Brand Name"
            },
            "productInclusions": [
              { "productId": "B0XXXXXXX1", "productIdType": "ASIN" },
              { "productId": "B0XXXXXXX2", "productIdType": "ASIN" },
              { "productId": "B0XXXXXXX3", "productIdType": "ASIN" }
            ],
            "landingPage": {
              "landingPageType": "ASIN_LIST"
            }
          }
        }
      }
    }
  ]
}
```

**Product count:** 3 minimum, 10 maximum.

**Landing page options for manual collections:**
- `ASIN_LIST` — Amazon creates a new landing page featuring your products
- `STORE` — links to your Brand Store page (provide `landingPageUrl`)

### 2.7 Create Targets

**Endpoint:** `POST /adsApi/v1/create/targets`

**Keyword target example:**

```json
{
  "targets": [
    {
      "adGroupId": "<ad-group-id>",
      "adProduct": "SPONSORED_BRANDS",
      "state": "ENABLED",
      "negative": false,
      "targetType": "KEYWORD",
      "bid": { "bid": 1.50 },
      "targetDetails": {
        "keywordTarget": {
          "keyword": "women running shoes",
          "matchType": "BROAD"
        }
      }
    }
  ]
}
```

**Product target example (manual collections only):**

```json
{
  "targets": [
    {
      "adGroupId": "<ad-group-id>",
      "adProduct": "SPONSORED_BRANDS",
      "state": "ENABLED",
      "negative": false,
      "targetType": "PRODUCT",
      "bid": { "bid": 2.00 },
      "targetDetails": {
        "productTarget": {
          "matchType": "PRODUCT_EXACT",
          "product": { "productId": "B0COMPETITOR" },
          "productIdType": "ASIN"
        }
      }
    }
  ]
}
```

**Match types available:**
- Keyword: `BROAD`, `PHRASE`, `EXACT`
- Product: `PRODUCT_EXACT`
- Theme: `KEYWORDS_RELATED_TO_YOUR_BRAND`, `KEYWORDS_RELATED_TO_YOUR_LANDING_PAGES`

### 2.8 Query (List) Ads

**Endpoint:** `POST /adsApi/v1/query/ads`

```json
{
  "adProductFilter": {
    "include": ["SPONSORED_BRANDS"]
  },
  "stateFilter": {
    "include": ["ENABLED", "PAUSED"]
  },
  "maxResults": 100
}
```

**Identifying SBC ads in response:** Look at the creative type:
- `autoCollectionSettings` present → Automatic Collection
- `manualCollectionSettings` present → Manual Collection
- `productCollectionSettings` present → Legacy Product Collection

### 2.9 Update Ads

**Endpoint:** `POST /adsApi/v1/update/ads`

```json
{
  "ads": [
    {
      "adId": "<ad-id>",
      "state": "PAUSED"
    }
  ]
}
```

**Updatable fields:**
- `state` (ENABLED/PAUSED)
- `name`
- `creative` (product inclusions for manual, product exclusions for auto, brand name, brand logo)

### 2.10 Delete Ads

**Endpoint:** `POST /adsApi/v1/delete/ads`

```json
{
  "adIds": ["<ad-id-1>", "<ad-id-2>"]
}
```

### 2.11 Bid Optimization

**Campaign-level bid settings** (via create or update campaign):

```json
{
  "optimizations": {
    "bidSettings": {
      "bidStrategy": "MANUAL",
      "bidAdjustments": {
        "placementBidAdjustments": [
          { "placement": "TOP_OF_SEARCH", "percentage": 50 },
          { "placement": "REST_OF_SEARCH", "percentage": 20 }
        ]
      }
    }
  }
}
```

**Bid strategies:**
- `MANUAL` — uses exact bid + placement adjustments
- `SALES_UP_AND_DOWN` — dynamic bidding ±100% (placement adjustments NOT supported)

## 3. API Response Patterns

### Multi-Status Response (207)

All create/update/delete endpoints return a 207 multi-status response:

```json
{
  "success": [
    {
      "index": 0,
      "ad": { /* full ad object */ }
    }
  ],
  "error": [
    {
      "index": 1,
      "errors": [
        {
          "code": "FIELD_VALUE_IS_INVALID",
          "message": "Product count must be between 3 and 10"
        }
      ]
    }
  ]
}
```

### Pagination

Query endpoints use token-based pagination:

```json
{
  "nextToken": "<opaque-token>",
  "ads": [ /* results */ ]
}
```

Pass `nextToken` in subsequent requests to get the next page.

### Common Error Codes

| Code | Meaning | Resolution |
|------|---------|------------|
| `BAD_REQUEST` | Invalid request schema | Check request body against schema |
| `FIELD_VALUE_IS_INVALID` | Invalid field value | Validate enum values, ASIN format |
| `FIELD_VALUE_IS_NULL` | Missing required field | Add required fields |
| `ACTIVE_RESOURCE_LIMIT_EXCEEDED` | Too many live resources | Archive unused resources |
| `ARCHIVED_PARENT_CANNOT_CREATE` | Parent is archived | Use active campaign/ad group |
| `TOO_MANY_REQUESTS` | Rate limited | Implement exponential backoff |
| `UNAUTHORIZED` | Invalid/expired token | Refresh OAuth token |
| `FORBIDDEN` | Insufficient permissions | Check OAuth scopes and account access |

## 4. Migrating from SB v4 API (`/sb/v4/`) to Unified API (`/adsApi/v1/`)

If you are currently using the **Sponsored Brands v4 OpenAPI** (`sponsored-brands-4-openapi.json`, paths like `/sb/v4/...`), and want to migrate to the **SB Unified API** (`AmazonAdsAPISBMerged_prod_3p.json`, paths like `/adsApi/v1/...`), this section covers all key differences.

### Confidence Legend

This section contains mappings derived from comparing two API schemas. Each mapping is labeled with a confidence level:

- ✅ **Confirmed from schema** — directly verifiable from API spec field names, types, and enum values
- ⚠️ **Reasonable inference** — based on semantic analysis, naming patterns, and functional equivalence; not from an official migration guide
- ❓ **Unconfirmed** — cannot be determined from schema alone; recommend opening a support case with Amazon Ads API team

### 4.1 Endpoint Path Mapping

| Operation | SB v4 API Path | Unified API Path |
|-----------|---------------|-----------------|
| Create Campaign | `POST /sb/v4/campaigns` | `POST /adsApi/v1/create/campaigns` |
| Update Campaign | `PUT /sb/v4/campaigns` | `POST /adsApi/v1/update/campaigns` |
| List Campaigns | `POST /sb/v4/campaigns/list` | `POST /adsApi/v1/query/campaigns` |
| Delete Campaigns | `POST /sb/v4/campaigns/delete` | `POST /adsApi/v1/delete/campaigns` |
| Create Ad Group | `POST /sb/v4/adGroups` | `POST /adsApi/v1/create/adGroups` |
| Update Ad Group | `PUT /sb/v4/adGroups` | `POST /adsApi/v1/update/adGroups` |
| List Ad Groups | `POST /sb/v4/adGroups/list` | `POST /adsApi/v1/query/adGroups` |
| Delete Ad Groups | `POST /sb/v4/adGroups/delete` | `POST /adsApi/v1/delete/adGroups` |
| Create Auto Collection Ad | `POST /sb/v4/ads/autoCollection` | `POST /adsApi/v1/create/ads` |
| Create Manual Collection Ad | `POST /sb/v4/ads/manualCollection` | `POST /adsApi/v1/create/ads` |
| Update Ad Creative (Auto) | `POST /sb/ads/creatives/autoCollection` | `POST /adsApi/v1/update/ads` |
| Update Ad Creative (Manual) | `POST /sb/ads/creatives/manualCollection` | `POST /adsApi/v1/update/ads` |
| Update Ad State | `PUT /sb/v4/ads` | `POST /adsApi/v1/update/ads` |
| List Ads | `POST /sb/v4/ads/list` | `POST /adsApi/v1/query/ads` |
| Delete Ads | `POST /sb/v4/ads/delete` | `POST /adsApi/v1/delete/ads` |
| Create Targets | *(separate targeting API)* | `POST /adsApi/v1/create/targets` |
| List Targets | *(separate targeting API)* | `POST /adsApi/v1/query/targets` |
| Update Targets | *(separate targeting API)* | `POST /adsApi/v1/update/targets` |
| Delete Targets | *(separate targeting API)* | `POST /adsApi/v1/delete/targets` |

**Key difference:** In v4, auto/manual collection ads have **separate creation endpoints**. In the Unified API, all ad types use a **single endpoint** (`/adsApi/v1/create/ads`) and the collection type is determined by the creative settings in the request body.

### 4.2 HTTP Method Changes

| Operation | SB v4 | Unified API |
|-----------|-------|------------|
| Update Campaign | `PUT` | `POST` |
| Update Ad Group | `PUT` | `POST` |
| Update Ad | `PUT` (state) + `POST` (creative) | `POST` (unified) |

The Unified API uses `POST` for all operations (create, query, update, delete).

### 4.3 Header Differences

| Header | SB v4 API | Unified API |
|--------|-----------|------------|
| Client ID | `Amazon-Advertising-API-ClientId` | `Amazon-Ads-ClientId` |
| Profile/Scope | `Amazon-Advertising-API-Scope` (required) | `Amazon-Advertising-API-Scope` (optional) |
| Account ID | Not used | `Amazon-Ads-AccountId` (optional) |
| Content-Type | Versioned: `application/vnd.sbcampaignresource.v4+json`, `application/vnd.sbadresource.v4+json`, etc. | Standard: `application/json` |

**Important:** The v4 API uses **versioned content types** (different per resource). The Unified API uses plain `application/json` for all requests.

### 4.4 Campaign Creation — Request Body Comparison

**SB v4 API:**
```json
{
  "campaigns": [
    {
      "name": "My Campaign",
      "state": "ENABLED",
      "budget": 50.00,
      "budgetType": "DAILY",
      "startDate": "2026-02-01",
      "brandEntityId": "<brand-entity-id>",
      "goal": "PAGE_VISIT",
      "bidding": {
        "bidOptimization": false,
        "bidAdjustmentsByPlacement": [
          { "placement": "TOP_OF_SEARCH", "percentage": 50 }
        ]
      }
    }
  ]
}
```

**Unified API:**
```json
{
  "campaigns": [
    {
      "adProduct": "SPONSORED_BRANDS",
      "name": "My Campaign",
      "state": "ENABLED",
      "costType": "CPC",
      "marketplaceScope": "SINGLE_MARKETPLACE",
      "startDateTime": "2026-02-01T00:00:00Z",
      "budgets": [
        {
          "budgetType": "MONETARY",
          "budgetValue": {
            "monetaryBudgetValue": {
              "monetaryBudget": { "value": 50.00 }
            }
          },
          "recurrenceTimePeriod": "DAILY"
        }
      ],
      "optimizations": {
        "bidSettings": {
          "bidStrategy": "MANUAL",
          "bidAdjustments": {
            "placementBidAdjustments": [
              { "placement": "TOP_OF_SEARCH", "percentage": 50 }
            ]
          }
        }
      }
    }
  ]
}
```

**Key differences:**
| Field | SB v4 | Unified API | Confidence |
|-------|-------|------------|------------|
| Ad product identifier | Not needed (implicit from path) | `adProduct: "SPONSORED_BRANDS"` (required) | ✅ Confirmed from schema |
| Budget | Flat `budget` + `budgetType` | Nested `budgets[].budgetValue.monetaryBudgetValue` | ✅ Confirmed from schema |
| Budget recurrence | `budgetType: "DAILY"` | `recurrenceTimePeriod: "DAILY"` | ✅ Confirmed from schema |
| Date format | `YYYY-MM-DD` string | ISO 8601 datetime `YYYY-MM-DDTHH:mm:ssZ` | ✅ Confirmed from schema |
| Date field names | `startDate` / `endDate` | `startDateTime` / `endDateTime` | ✅ Confirmed from schema |
| Bidding | `bidding.bidOptimization` + `bidAdjustmentsByPlacement` | `optimizations.bidSettings.bidStrategy` + `bidAdjustments.placementBidAdjustments` | ✅ Confirmed from schema |
| Bid optimization | `bidOptimization: true/false` | `bidStrategy: "SALES_UP_AND_DOWN"` or `"MANUAL"` | ⚠️ Reasonable inference — both have two-state toggle with same semantic |
| Placement enum values | `HOME`, `DETAIL_PAGE`, `OTHER`, `TOP_OF_SEARCH` | `HOME_PAGE`, `PRODUCT_PAGE`, `REST_OF_SEARCH`, `TOP_OF_SEARCH` | ⚠️ Reasonable inference — based on semantic matching of enum values |
| Brand ID | `brandEntityId` | `brandId` | ✅ Confirmed from schema |
| Goal | `goal: "PAGE_VISIT"` / `"BRAND_IMPRESSION_SHARE"` | `optimizations.goalSettings.kpi: "CLICKS"` / `"TOP_OF_SEARCH_IMPRESSION_SHARE"` | ⚠️ Reasonable inference — see FAQ section for details |
| Marketplace | Not in campaign body | `marketplaceScope: "SINGLE_MARKETPLACE"` (required) | ✅ Confirmed from schema |
| Cost type | `costType: "CPC"` (optional) | `costType: "CPC"` (required) | ✅ Confirmed from schema |

> ⚠️ Items marked "Reasonable inference" are based on semantic analysis of both API schemas. They are NOT from an official migration document. If any of these mappings cause issues, open a support case with the Amazon Ads API team.

### 4.5 Ad Creation — Auto Collection Comparison

**SB v4 API** (`POST /sb/v4/ads/autoCollection`):
```json
{
  "ads": [
    {
      "adGroupId": "<ad-group-id>",
      "name": "Auto Collection Ad",
      "state": "ENABLED",
      "creative": {
        "brandName": "Your Brand",
        "brandLogoAssetID": "<optional-asset-id>",
        "asinExclusions": ["B0XXXXXXX1"]
      }
    }
  ]
}
```

**Unified API** (`POST /adsApi/v1/create/ads`):
```json
{
  "ads": [
    {
      "adGroupId": "<ad-group-id>",
      "adProduct": "SPONSORED_BRANDS",
      "adType": "COMPONENT",
      "name": "Auto Collection Ad",
      "state": "ENABLED",
      "creative": {
        "componentCreative": {
          "autoCollectionSettings": {
            "sharedSettings": {
              "brand": "Your Brand",
              "brandLogos": {
                "assetId": "<optional-asset-id>",
                "assetVersion": "<asset-version>"
              }
            },
            "productExclusions": [
              { "productId": "B0XXXXXXX1", "productIdType": "ASIN" }
            ]
          }
        }
      }
    }
  ]
}
```

**Key differences:**
| Field | SB v4 | Unified API |
|-------|-------|------------|
| Endpoint | Dedicated `/sb/v4/ads/autoCollection` | Shared `/adsApi/v1/create/ads` |
| Ad product | Not needed | `adProduct: "SPONSORED_BRANDS"` required |
| Ad type | Not needed | `adType: "COMPONENT"` required |
| Creative wrapper | `creative.brandName` (flat) | `creative.componentCreative.autoCollectionSettings.sharedSettings.brand` (nested) |
| Brand logo | `brandLogoAssetID` (string) | `brandLogos: { assetId, assetVersion }` (object) |
| Brand logo crop | `brandLogoCrop: { top, left, width, height }` | `brandLogos.formatProperties: [{ top, left, width, height }]` |
| ASIN exclusions | `asinExclusions: ["B0XX"]` (string array) | `productExclusions: [{ productId, productIdType }]` (object array) |
| Max exclusions | 1000 | 1000 |

### 4.7 Ad Update / Creative Revision Comparison

**SB v4 API** has separate flows:
- State/name update: `PUT /sb/v4/ads` with `{ ads: [{ adId, state }] }`
- Creative revision: `POST /sb/ads/creatives/autoCollection` or `/manualCollection`

**Unified API** unifies both into a single endpoint:
- `POST /adsApi/v1/update/ads` with `{ ads: [{ adId, state, creative: {...} }] }`

### 4.8 List/Query Differences

**SB v4 API** (`POST /sb/v4/ads/list`):
```json
{
  "campaignIdFilter": { "include": ["<id>"] },
  "stateFilter": { "include": ["ENABLED", "PAUSED"] },
  "maxResults": 100,
  "nextToken": "..."
}
```

**Unified API** (`POST /adsApi/v1/query/ads`):
```json
{
  "adProductFilter": { "include": ["SPONSORED_BRANDS"] },
  "campaignIdFilter": { "include": ["<id>"] },
  "stateFilter": { "include": ["ENABLED", "PAUSED"] },
  "maxResults": 100,
  "nextToken": "..."
}
```

**Key difference:** The Unified API requires `adProductFilter` since the endpoint is shared across ad products.

### 4.9 Response Format Differences

**SB v4 success response:**
```json
{
  "campaigns": {
    "success": [
      { "campaignId": "123", "index": 0, "campaign": {...} }
    ],
    "error": [
      { "index": 1, "errors": [...] }
    ]
  }
}
```

**Unified API success response:**
```json
{
  "success": [
    { "index": 0, "campaign": {...} }
  ],
  "error": [
    { "index": 1, "errors": [...] }
  ]
}
```

The Unified API returns `success`/`error` at the top level without the resource-type wrapper.

### 4.10 Features NOT Available in Unified API

The following v4 features are not in the Unified API spec and must still use v4 endpoints:

- Budget rules (`/sb/budgetRules`)
- Budget recommendations (`/sb/campaigns/budgetRecommendations`)
- Budget usage (`/sb/campaigns/budget/usage`)
- Campaign insights (`/sb/campaigns/insights`)
- Forecasts (`/sb/forecasts`)
- Optimization rules (`/sb/rules/optimization`)
- Headline recommendations (`/sb/recommendations/creative/headline`)
- Negative target brand recommendations (`/sb/negativeTargets/brands/recommendations`)
- Product targeting categories (`/sb/targets/categories`)
- V3 campaign migration (`/sb/v4/legacyCampaigns/migrationJob`)

### 4.11 Migration Checklist

- [ ] Update header names (`Amazon-Advertising-API-ClientId` → `Amazon-Ads-ClientId`)
- [ ] Change Content-Type from versioned to `application/json`
- [ ] Change all HTTP methods to `POST` (no more `PUT`)
- [ ] Update endpoint paths (`/sb/v4/...` → `/adsApi/v1/...`)
- [ ] Add `adProduct: "SPONSORED_BRANDS"` to all create/query requests
- [ ] Add `adType: "COMPONENT"` to ad creation requests
- [ ] Add `marketplaceScope: "SINGLE_MARKETPLACE"` to campaign creation
- [ ] Restructure budget from flat `budget`+`budgetType` to nested `budgets[]` array
- [ ] Convert date strings from `YYYY-MM-DD` to ISO 8601 datetime

## 6. Frequently Asked Questions

### General

**Q: What is the minimum number of ASINs required?**
A: Manual collections require 3–10 ASINs. Automatic collections don't require ASIN input (AI selects them).

**Q: Can I control which ASINs appear in auto collections?**
A: You can exclude up to 100 ASINs via `productExclusions`. You cannot force specific ASINs to appear.

**Q: Can I set a custom headline?**
A: No. Headlines are auto-generated by Amazon AI based on products and category. You cannot preview or edit them.

**Q: Are drafts supported for SBC ads?**
A: No. SBC ads do not support drafts.

**Q: Can I convert between manual and auto collections?**
A: No. You must create a new ad group with the desired collection type.

**Q: Can I create multiple ads in an auto collection ad group?**
A: No. Only one ad per ad group for automatic collections. Manual collections allow multiple ads.

### Targeting

**Q: What targeting is available for each type?**
A: Auto collections support keyword targeting only. Manual collections support both keyword and product targeting.

**Q: Can I use negative targeting?**
A: Yes, both collection types support negative keyword targeting.

**Q: Does targeting affect which ASINs are shown in auto collections?**
A: Yes. AI uses your keyword targets and shopper queries to determine which products to surface.

### Landing Pages

**Q: What landing page is used for auto collections?**
A: A dynamically generated landing page created by Amazon featuring products from the ad and related products.

**Q: What are the landing page options for manual collections?**
A: Brand Store page (including subpages) or a new landing page created by Amazon.

**Q: Where do product clicks go?**
A: All individual product clicks go to the respective Product Detail Page regardless of collection type.

### Campaign Management

**Q: What can I edit after creation?**

| Edit | Manual | Auto |
|------|--------|------|
| State (pause/enable) | ✅ | ✅ |
| Products | ✅ | ❌ (exclusions only) |
| Brand name | ✅ | ✅ |
| Brand logo | ✅ | ✅ |
| Landing page | ✅ | ❌ |
| Targets | ✅ | ✅ |
| Bids | ✅ | ✅ |
| Headline/Title | ❌ | ❌ |

**Q: How long does moderation take?**
A: Up to 72 hours. No edits possible during moderation.

**Q: How do I track performance?**
A: Use existing Sponsored Brands reporting — Campaign Performance, Search Term, Keyword, and Attributed Purchases reports. No changes to reporting API.

### API Migration (v4 → Unified API)

**Q: Can I use both APIs simultaneously?**
A: Yes. Existing v4 endpoints continue to work. You can migrate gradually — for example, use Unified API for new SBC campaigns while keeping v4 for budget rules and insights.

**Q: Why does the Unified API require `adProduct` everywhere?**
A: The Unified API is a unified API shared across ad products (Sponsored Brands, Sponsored Products, etc.). The `adProductFilter`/`adProduct` field tells the API which product you're working with.

**Q: I'm getting 400 errors after switching to the Unified API, what should I check?**
A: Common causes:
1. Missing `adProduct: "SPONSORED_BRANDS"` in request body
2. Missing `marketplaceScope: "SINGLE_MARKETPLACE"` in campaign creation
3. Wrong date format (use ISO 8601 `2026-01-15T00:00:00Z`, not `2026-01-15`)
4. Using old header name `Amazon-Advertising-API-ClientId` instead of `Amazon-Ads-ClientId`
5. Using wrong Content-Type (should be `application/json`, not versioned types)
6. Budget structured as flat number instead of nested `budgets[]` array

**Q: Where did `bidOptimization: true` go?**
A: In the Unified API, use `bidStrategy: "SALES_UP_AND_DOWN"` (equivalent to bidOptimization=true) or `bidStrategy: "MANUAL"` (equivalent to bidOptimization=false).

**Q: The v4 placement enum `OTHER` doesn't work in Unified API?**
A: Correct. Use these mappings: `HOME` → `HOME_PAGE`, `DETAIL_PAGE` → `PRODUCT_PAGE`, `OTHER` → `REST_OF_SEARCH`, `TOP_OF_SEARCH` stays the same.

**Q: Do I still need separate endpoints for auto vs manual collection ads?**
A: No. The Unified API uses a single endpoint `/adsApi/v1/create/ads`. The collection type is determined by whether you include `autoCollectionSettings` or `manualCollectionSettings` in the creative body.

**Q: How do I pass ASINs in the Unified API?**
A: Instead of a simple string array (`asins: ["B0XX"]`), use object arrays: `productInclusions: [{ "productId": "B0XX", "productIdType": "ASIN" }]`. Same for exclusions.

### Campaign-Level Migration Issues

**Q: How does v4 `goal` map to adsApi `goalSettings.kpi`?**

Based on the API schema definitions:
- v4 `goal` has: `PAGE_VISIT` (default), `BRAND_IMPRESSION_SHARE`
- adsApi `goalSettings.kpi` has: `CLICKS`, `TOP_OF_SEARCH_IMPRESSION_SHARE`

**Reasonable inference (based on semantic and costType correlation):**

| v4 `goal` | adsApi `goalSettings.kpi` | costType | Reasoning |
|-----------|--------------------------|----------|-----------|
| `PAGE_VISIT` | `CLICKS` | `CPC` | Both target driving traffic/clicks |
| `BRAND_IMPRESSION_SHARE` | `TOP_OF_SEARCH_IMPRESSION_SHARE` | `VCPM` | Both target maximizing top-of-search impression share |

Additionally, adsApi's `goalSettings` in response contains both `goal` (enum: `AWARENESS`, `CONSIDERATION`, `CONVERSIONS`) and `kpi`. The inferred full mapping:

| v4 goal | adsApi goalSettings.goal | adsApi goalSettings.kpi |
|---------|--------------------------|------------------------|
| `PAGE_VISIT` | `CONVERSIONS` | `CLICKS` |
| `BRAND_IMPRESSION_SHARE` | `AWARENESS` | `TOP_OF_SEARCH_IMPRESSION_SHARE` |

When creating campaigns, `SBCreateGoalSettings` only requires `kpi` (not `goal`).

> ⚠️ **Note:** This mapping is a reasonable inference based on semantic analysis of both API specs and the costType linkage. It is NOT explicitly documented in a migration guide. If you need official confirmation, **please open a support case** with Amazon Ads API support team to verify the exact mapping.

**Q: How does v4 `shopperCohortBidAdjustments` map to adsApi `audienceBidAdjustments`?**

**Confirmed from schema:**

v4 structure:
```json
"bidding": {
  "shopperCohortBidAdjustments": [{
    "shopperCohortType": "AUDIENCE_SEGMENT",
    "percentage": 50,
    "audienceSegments": [{ "audienceId": "xxx", "audienceSegmentType": "SPONSORED_ADS_AMC" }]
  }]
}
```

adsApi structure:
```json
"optimizations": {
  "bidSettings": {
    "bidAdjustments": {
      "audienceBidAdjustments": [{ "audienceId": "xxx", "percentage": 50 }]
    }
  }
}
```

**What's confirmed:**
- adsApi `audienceBidAdjustments` only has two fields: `audienceId` (string, required) and `percentage` (int32, required)
- There is NO `audienceSegmentType` field in the adsApi schema
- There is NO `shopperCohortType` field in the adsApi schema
- adsApi also has a `shopperSegmentBidAdjustments` field but it is explicitly marked as **"Legacy SB field (marked for deprecation)"** — do NOT use it

**Reasonable inference:**
- `shopperCohortType: "AUDIENCE_SEGMENT"` is implicitly fixed in adsApi — no need to pass it
- `audienceSegmentType` (SPONSORED_ADS_AMC / BEHAVIOR_DYNAMIC) distinction is likely handled server-side based on the `audienceId` itself — no need to explicitly declare the type

> ⚠️ **Unconfirmed (open a support case to verify):**
> - How "default audience" is expressed in adsApi — the schema does not reveal this
> - Whether all `audienceId` values from v4 work identically in adsApi
> - Whether there are any behavioral differences for SPONSORED_ADS_AMC vs BEHAVIOR_DYNAMIC audiences when the type is not explicitly provided
>
> **Recommendation:** Open a support case to confirm:
> 1. Whether `audienceId` values are directly portable from v4 to adsApi
> 2. How "default audience" concept maps in adsApi
> 3. Whether the removal of `audienceSegmentType` has any impact on bid adjustment behavior

### Errors & Troubleshooting

**Q: My ad isn't getting impressions.**
A: Check: bid amounts, budget, targeting settings, ensure enough eligible in-stock products (min 3), check campaign/ad approval status.

**Q: Product eligibility issues.**
A: Verify ASINs are: active in the marketplace, in stock, owned by the advertiser account, and not restricted.

**Q: Rate limiting (429 errors).**
A: Implement exponential backoff. Respect rate limits per endpoint.

**Q: Content too large (413 error).**
A: Reduce batch size. Max batch sizes:
- Campaigns: 10
- Ad Groups: 10
- Ads: 10
- Targets: 1000

## 8. Complete Campaign Setup Flow (API)

